# 校园二手书交易平台 - MVP 设计

## 一、项目目标
打造一个校内二手书信息聚合平台，解决当前信息分散、难以检索的问题。核心价值在于：学生可**自由发布**闲置书籍，也能**快速检索**他人发布的书籍，并通过预留联系方式自行完成线下交易。**V1.0 版本不涉及支付、物流或站内即时通讯。**

## 二、功能模块
1. **首页 / 列表页**  
   - 顶部提供搜索框，支持按书名模糊搜索。  
   - 下方以列表或卡片形式展示所有在售书籍，包含书名、售价、卖家昵称、发布时间，并按发布时间倒序排列。

2. **发布页**  
   - 表单字段：书名（必填）、售价（必填）、原价（选填）、新旧程度描述（选填）、联系方式（必填，如手机号/微信号）、书籍图片（选填，支持上传一张）。

3. **详情页**  
   - 展示书籍完整信息，并提供“一键复制联系方式”功能，方便买家快速联系卖家。

## 三、数据表设计（SQLite）

**用户表（users）**  
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| student_id | TEXT | 学号，用作登录账号 |
| password_hash | TEXT | 存储 bcrypt 哈希后的密码 |
| nickname | TEXT | 用户昵称 |
| created_at | DATETIME | 注册时间 |

**书籍表（books）**  
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| user_id | INTEGER | 外键，关联 users.id |
| title | TEXT | 书名 |
| price | REAL | 售价 |
| original_price | REAL | 原价（选填） |
| description | TEXT | 书籍新旧程度或使用情况描述 |
| contact | TEXT | 联系方式（手机/微信） |
| image_url | TEXT | 图片存储路径（选填） |
| status | INTEGER | 状态，1 表示在售，0 表示已下架 |
| created_at | DATETIME | 发布时间 |

## 四、后端接口（API）

| 方法 | 路径 | 说明 | 需登录 |
|------|------|------|--------|
| POST | `/api/register` | 用户注册，密码需经 bcrypt 哈希处理 | 否 |
| POST | `/api/login` | 用户登录，返回 session 或 token | 否 |
| POST | `/api/books` | 发布新书籍 | 是 |
| GET | `/api/books` | 获取书籍列表，支持 `?keyword=` 模糊搜索 | 否 |
| GET | `/api/books/:id` | 获取指定书籍的详情 | 否 |

## 五、安全规范（必须执行）

1. **密码保护**：后端接收明文密码后，必须使用 `bcrypt` 进行加盐哈希，数据库中**只存储哈希值**，严禁存储明文密码。  
2. **环境变量管理**：所有敏感配置（如 `SECRET_KEY`、数据库路径等）必须写入项目根目录的 `.env` 文件中，并在 `.gitignore` 中明确忽略该文件，防止泄露。  
3. **Git 提交规范**：**严禁**将 `.env` 文件或任何包含敏感信息（如硬编码密码）的代码提交至 GitHub。如已误提交，需立即使用 `git filter-repo` 或重置仓库等方式清除历史记录，确保安全。

## 六、技术选型

- **后端**：Python Flask 或 Node.js Express（二选一，根据团队熟悉程度决定）  
- **数据库**：SQLite3（无需额外安装，便于快速开发和演示）  
- **前端**：HTML + CSS + 原生 JavaScript，或使用 Vue.js CDN 模式；优先做好移动端适配，因为学生多使用手机访问。
